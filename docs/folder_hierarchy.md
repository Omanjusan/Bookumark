# フォルダ階層・移動モード設計

策定日: 2026-07-21

## 1. 目的

Firefox公式ブックマークDBを正本として尊重しつつ、Bookumark内の表示順と公式フォルダ階層の編集を安全に両立する。

本書では、フォルダとブックマークの表示上の扱い、現在フォルダ、パンくずデータ、フォルダ単位のカスタム順、移動モード、公式DB編集の安全境界および実装フェーズを定義する。

favicon取得は本設計より優先度を下げ、別フェーズへ保留する。

## 2. 判断基準

「正しさ」を次の3種類へ分離する。

| 種類 | 意味 |
| --- | --- |
| データの正しさ | Firefox公式DBのフォルダ階層と項目を壊さない |
| 表示の正しさ | 現在の操作目的に必要な要素を分かりやすく表示する |
| 指標の正しさ | 同じ意味で比較できる値だけを同一スケールへ反映する |

公式DBに存在するすべてのノードを、同じ見た目や同じランキング母集団へ入れることは要求しない。正本の保持と表示形式は別の責務として扱う。

## 3. フォルダとブックマークの表示方針

### 3.1 基本方針

- 現在フォルダの直下にある項目だけを対象とする。
- フォルダは固定サイズのフォルダ領域へ表示する。
- URLを持つブックマークはアイテム領域へ表示する。
- フォルダはブックマークの訪問回数、順位、パーセンタイルの母集団`N`へ含めない。
- フォルダ領域とブックマーク領域は別の表示集合として生成する。
- フォルダの初期表示順はFirefox公式順とする。
- フォルダへのdropによる公式DB変更は、通常時およびカスタム配置時には行わない。

フォルダにはブックマークと同じ訪問回数が存在しない。欠損値を訪問回数`0`として扱わず、フォルダ数によってブックマークの順位やサイズが変化しないようにする。

### 3.2 将来のフォルダ指標

フォルダをスケール表示する場合は、ブックマークの訪問回数とは別の集計指標として定義する。

- 配下ブックマークの訪問回数合計
- 配下で最後に訪問された日時
- 配下ブックマーク数
- Bookumark内でフォルダを開いた回数

これらは「フォルダ活動度」などの独立した軸とし、ブックマーク自身の訪問回数ランキングへ無条件に混在させない。

## 4. 現在フォルダ

- 初回起動時はFirefoxのブックマークルートを現在フォルダとする。
- 2回目以降は前回表示していたフォルダを復元する。
- 前回フォルダが削除されていた場合は、存在する最寄りの親フォルダへ戻る。
- 親フォルダも特定できない場合はルートへ戻る。
- Firefoxのツールバー、メニュー、未整理、モバイルなどの特殊フォルダも、ナビゲーション上は通常のフォルダとして表示する。
- 特殊フォルダ自体を公式DB内で移動する操作は許可しない。

検索範囲は初期版では現在フォルダ直下だけとする。全階層検索は後続機能とする。

## 5. パンくずデータ契約

パンくずはグリッド内へ描画しない。ドッキングベイ側が任意にチップ表示できるよう、ルートから現在フォルダまでの配列だけを提供する。

```ts
type BreadcrumbItem = {
  guid: string;
  title: string;
};

function buildBreadcrumb(
  currentFolderGuid: string,
  foldersByGuid: ReadonlyMap<string, FolderItem>,
): BreadcrumbItem[];
```

契約:

- ルートから現在フォルダまでの順序で返す。
- 現在フォルダ自身を含める。
- DOM、チップ、CSSおよびグリッド寸法へ依存しない。
- 親参照が欠損していても無限ループしない。
- 循環した親参照を検出し、有限時間で終了する。
- パンくずの表示・非表示はドッキングベイ側の責務とする。

パンくずはグリッド外の部品であるため、ブックマークグリッドの可処分領域および面積予算を増減させない。

## 6. フォルダ単位のカスタム順

カスタム順はフォルダ単位で保持する。異なる親フォルダの項目を1本の順序配列へ混在させない。

```ts
type CustomOrderByFolder = Record<string, string[]>;
```

- キーは親フォルダGUIDとする。
- 値はそのフォルダ直下にある表示対象GUIDの順序とする。
- 順序に存在しない新規項目はFirefox公式順で末尾へ追加する。
- 削除・移動済みのGUIDはreconcile時に除去する。
- 現在の全体カスタム順は、各項目の親フォルダごとに分割して移行する。
- 移行後も各フォルダ内で既存の相対順を可能な限り維持する。

## 7. 移動モード

### 7.1 状態モデル

2つの独立booleanは使用しない。UI上は3択のセグメントコントロールとし、内部状態も排他的な列挙型とする。

```ts
type MovementMode =
  | "custom-order"
  | "normal"
  | "directory-move";
```

推奨表示:

```text
[ カスタム配置 ] [ 通常 ] [ 公式整理 ]
```

| モード | D&Dの意味 | 書き込み先 |
| --- | --- | --- |
| `custom-order` | 現在フォルダ内の表示順変更 | `storage.local` |
| `normal` | D&D不可 | なし |
| `directory-move` | 公式順・公式フォルダ階層の編集 | Firefox bookmarks API |

中央の`normal`を安全な既定状態とする。左から右へ「Bookumark内だけの変更」「変更なし」「Firefox本体の変更」という意味を持たせる。

### 7.2 UI契約

- HTMLのradio groupをCSSでセグメント表示する。
- 同時に選択できる状態は必ず1つとする。
- 矢印キーで選択状態を変更できるようにする。
- スクリーンリーダーへ3択として伝える。
- `directory-move`の表示名は「公式整理」とする。
- 「Firefox本体へ反映」という補足を表示する。
- `custom-order`は通常のアクセント色、`normal`はグレー、`directory-move`はアンバー系とする。
- 公式DB操作失敗の通知だけをエラー色とする。

### 7.3 状態遷移

- 初期状態は`normal`とする。
- パネルの再読み込みおよび再起動後は`normal`へ戻す。
- 検索またはフィルタ開始時は`normal`へ戻す。
- 各モード間は直接切り替え可能とする。
- 公式DB移動成功後も`directory-move`を維持し、連続整理を可能にする。
- 公式DB更新失敗時はモードを維持し、項目表示だけを公式DBの状態へ戻す。

## 8. D&Dの意味

### 8.1 カスタム配置

- 現在フォルダ直下の表示順だけを変更する。
- Firefox公式DBの親フォルダやindexを変更しない。
- フォルダ単位のカスタム順へ保存する。
- 検索・フィルタ中は無効とする。

### 8.2 公式整理

- 項目間の境界へdropすると、同じ親フォルダ内の公式indexを変更する。
- フォルダ中央へdropすると、そのフォルダの子へ移動する。
- 先頭・末尾境界へdropすると、公式順の先頭・末尾へ移動する。
- フォルダだけでなく、URLを持つブックマークも別フォルダへ移動可能とする。
- 自分自身へのdropは無視する。
- フォルダを自身の子孫へ移動する操作は禁止する。
- Firefoxの特殊ルートフォルダ自体の移動は禁止する。
- 検索・フィルタ中は無効とする。
- フォルダ中央には「このフォルダへ移動」、境界には「ここへ並べ替え」に相当する視覚表示を出す。

公式整理はFirefox公式DBを書き換える。通常時やカスタム配置時のdropを公式整理として解釈してはならない。

## 9. 公式DB移動の安全性

移動前に元の位置を保持する。

```ts
type BookmarkMoveSnapshot = {
  guid: string;
  previousParentGuid: string;
  previousIndex: number;
};
```

- API成功後に公式DBの状態を再取得して表示を確定する。
- API失敗時は楽観表示を確定せず、公式DBの状態へ戻す。
- 成功時は移動元、移動先およびUndo操作を通知する。
- Undoは保持した親GUIDとindexを使って公式DBへ戻す。
- 移動先が操作中に削除された場合は失敗として扱い、現在状態を再取得する。
- 現在表示中のフォルダ自身が外部で移動された場合は、パンくずを再構築する。
- 公式移動後は、移動元と移動先のフォルダ単位カスタム順をreconcileする。

## 11. テスト観点

### 11.1 ツリーとナビゲーション

- ルート直下のフォルダとブックマークを分離できる。
- 空フォルダを現在フォルダとして表示できる。
- 多段の入れ子フォルダを移動できる。
- 前回フォルダが存在すれば復元する。
- 前回フォルダが削除済みなら親またはルートへ戻る。
- 親参照の欠損と循環で処理が停止しない。

### 11.2 表示集合

- フォルダ数が変化してもブックマークの順位と希望サイズが変わらない。
- ブックマーク0件、フォルダ1件以上でフォルダ領域と空のアイテム状態を表示できる。
- フォルダ0件、ブックマーク1件以上で通常グリッドを表示できる。
- 検索対象は現在フォルダ直下のブックマークだけになる。

### 11.3 カスタム順移行

- 既存全体順を親フォルダ別へ分割する。
- 同じ親を持つ項目の相対順を維持する。
- 新規項目をFirefox公式順で末尾へ追加する。
- 削除・移動済み項目を各フォルダ順から除去する。
- 移行を再実行しても結果が変化しない。

### 11.4 移動モード

- 常に1モードだけが選択される。
- 初期化と再読み込みで`normal`になる。
- 検索・フィルタ開始時に`normal`になる。
- `custom-order`で公式DBを変更しない。
- `directory-move`でstorage.local順だけを変更しない。
- 通常クリックとD&Dが競合しない。

### 11.5 公式整理

- 同一親内で先頭、中央、末尾へ移動できる。
- ブックマークを別フォルダへ移動できる。
- フォルダを別フォルダへ移動できる。
- 自分自身と子孫へのフォルダ移動を拒否する。
- 特殊ルートフォルダの移動を拒否する。
- API失敗後に公式DBと表示が一致する。
- Undoで元の親とindexへ戻る。
- 移動元・移動先のカスタム順が再整合される。

## 12. 完了条件

- 既存の入れ子フォルダを閲覧できる。
- 現在フォルダとパンくず配列が一貫する。
- フォルダがブックマークのスケール母集団へ混入しない。
- カスタム順がフォルダ単位で保存される。
- 3つの移動モードが視覚・内部状態ともに排他的である。
- カスタム配置と公式整理が異なる保存先へ正しく反映される。
- 公式DBの危険な移動を拒否できる。
- 公式移動の失敗時に表示と正本が不一致にならない。
- 公式移動をUndoできる。
- 単体テスト、統合テスト、Firefox実機確認が成功する。
