# パネル表示モード 詳細設計

策定日: 2026-07-21

## 1. 目的

パネル表示モードは、現在の表示集合におけるソート順位をタイルの面積へ反映し、重要なブックマークを視覚的に強調する表示形式である。

本書では、検索・フィルタ・ソート・自由移動との入出力契約、サイズ配分、グリッド描画、再計算、操作、状態表示およびテストケースを定義する。検索、ソート、自由移動そのものの内部設計は対象外とする。

## 2. スコープ

### 2.1 対象

- URLを持つブックマークのパネル表示
- 検索・フィルタ後の表示集合を対象とした順位計算
- ソート軸のスケーラビリティに応じたサイズ切り替え
- 7段階のタイルサイズ
- 幅制約と面積予算によるカスケード降格
- グリッドコンテナのリサイズに応じた再計算
- サイズ段階に応じた情報量の切り替え
- 読み込み中、空、エラー状態の表示
- 左クリックによる新しいアクティブタブでの遷移
- 最大2,000件を想定した処理

### 2.2 対象外・TODO

- 表示モード選択状態の永続化
- フォルダ、セパレーターおよび階層ナビゲーション
- 自由移動中に並び順とは別のスケール軸を選ぶ機能
- キーボード操作
- 中クリック、Ctrl/Cmdクリックなどの修飾キー操作
- タッチ端末対応
- 極小タイルの操作領域改善
- ツールチップ、詳細ポップオーバー、ホバー拡大
- 配置およびサイズ変更のアニメーション
- `prefers-reduced-motion`対応
- 仮想スクロールおよび段階描画
- Firefox自動E2E
- エラー状態からの再試行UI
- favicon欠損時の文字・色による識別（初期版は共通アイコン）
- 自動彩色
  - 単一色とスケール段階を組み合わせたグラデーション
  - 色相・彩度・明度にガードレールを設けたパステルランダムパネル
  - 配色の決定性、コントラスト、テーマ対応および方式切り替え

## 3. 用語

| 用語 | 定義 |
| --- | --- |
| 表示集合 | 検索およびフィルタを適用した後、パネルに渡されるブックマーク集合 |
| 表示順 | 選択中のソート軸と方向を適用した後の順序 |
| スケーラブル軸 | 順位をタイルサイズへ反映できるソート軸 |
| 希望サイズ | 相対順位だけから求めた、降格前のサイズ段階 |
| 確定サイズ | 幅制約と面積予算を適用した後のサイズ段階 |
| 面積予算 | 各サイズ群がグリッドの表示領域に対して占有できる累積面積上限 |
| 等倍 | サイズ`1`、グリッド上の`4x4`セル |
| 降格 | 希望サイズから1段階小さいサイズへ移す処理 |

## 4. ソート軸との契約

パネル側は訪問回数などの軸名を直接判定しない。ソート機能から、少なくとも次の情報を受け取る。

```ts
type SortAxis<T> = {
  id: string;
  scalable: boolean;
  direction: "asc" | "desc";
  valueOf(item: BookmarkItem): T | null | undefined;
  compare(a: BookmarkItem, b: BookmarkItem): number;
  formatValue?(value: T): string;
};
```

- `scalable: true`の場合、表示順における順位をサイズへ反映する。
- `scalable: false`の場合、値や方向に関係なく全項目へ等倍を適用する。
- スケーラブル軸の欠損値は、その軸の最低値としてソートする。
- サイズはスカラー値の絶対量ではなく、方向適用後の表示順位を表す。
  - 降順では、大きい値から始まる表示順の先頭ほど大きくする。
  - 昇順では、小さい値から始まる表示順の先頭ほど大きくする。
  - 訪問回数昇順では、未訪問を含む低訪問項目が大きくなる。
- 同値項目は、ソート処理から渡された現在の表示順を維持する。パネル側で第2ソートを行わない。
- 実装上の再現性が必要な場合、GUIDは順序を変更しない安定キーとしてのみ利用する。

### 4.1 自由移動との契約

初期版では、自由移動ON時のカスタム軸を`scalable: false`として扱い、全項目を等倍にする。

将来は「並び順の軸」と「サイズ算出軸」を分離し、自由移動中でもサイズ算出軸が`scalable: true`なら大小を維持できるよう、契約を拡張可能にしておく。初期版ではこの混在処理を実装しない。

## 5. タイルサイズ

### 5.1 サイズ段階

最小セルを基準に、正方形と横長を交互に配置する。面積は段階ごとに正確に2倍となる。

| 面積倍率 | グリッドspan | 形状 | 24pxセルでの外寸（gap除外） |
| ---: | ---: | --- | ---: |
| `1/16` | `1x1` | 正方形 | `24x24px` |
| `1/8` | `2x1` | 横長 | `48x24px` |
| `1/4` | `2x2` | 正方形 | `48x48px` |
| `1/2` | `4x2` | 横長 | `96x48px` |
| `1` | `4x4` | 正方形・等倍 | `96x96px` |
| `2` | `8x4` | 横長 | `192x96px` |
| `4` | `8x8` | 正方形・最大 | `192x192px` |

サイズ順は次の定数として一元管理する。

```text
4 -> 2 -> 1 -> 1/2 -> 1/4 -> 1/8 -> 1/16
```

- 最小セルは初期版では24px固定とする。
- CSSカスタムプロパティ（例: `--panel-grid-unit: 24px`）として分離する。
- セル寸法のレスポンシブ化やユーザー設定はTODOとする。
- `1/16`は無制限に受け入れられる最終降格先とする。

### 5.2 サイズ別の表示情報

| サイズ | 表示情報 |
| ---: | --- |
| `1/16` | faviconのみ |
| `1/8` | faviconのみ |
| `1/4` | favicon、短いタイトル1行 |
| `1/2` | favicon、タイトル最大2行 |
| `1` | favicon、タイトル、ドメイン |
| `2` | サイズ`1`の情報、スケール軸の表示値 |
| `4` | サイズ`2`の情報、最終訪問日時などの補助情報 |

- スケール軸の表示値は、軸が提供するフォーマッターを利用する。
- faviconを取得できない場合は共通ブックマークアイコンを表示する。
- 空URLや不正URLが入力に混在しても描画全体を停止させない。ただし初期版の正式な表示対象はURLを持つブックマークのみとする。

## 6. 希望サイズの計算

### 6.1 母集団

検索・フィルタ適用後の表示集合を母集団とする。検索語またはフィルタが変わるたびに、表示集合内で順位とサイズを再計算する。同じブックマークでも表示集合が変わればサイズが変わることを許容する。

### 6.2 パーセンタイル

ソート済みの0始まり順位を`i`、表示件数を`N`として、次で計算する。

```text
percentile = i / N
```

- `N = 0`では計算せず、空状態を表示する。
- `N = 1`では`percentile = 0`となり、最大サイズの候補になる。
- 同じ値や近似値であっても、順位境界をまたぐことによるサイズ差を許容する。

### 6.3 初期分位点

| パーセンタイル | 希望サイズ |
| ---: | ---: |
| `0 <= p < 0.05` | `4` |
| `0.05 <= p < 0.15` | `2` |
| `0.15 <= p < 0.30` | `1` |
| `0.30 <= p < 0.50` | `1/2` |
| `0.50 <= p < 0.70` | `1/4` |
| `0.70 <= p < 0.85` | `1/8` |
| `0.85 <= p < 1.00` | `1/16` |

分位点はUX調整値としてロジックから分離する。

## 7. 確定サイズの割り当て

### 7.1 グリッド表示領域

- 検索、フィルタ、ソート、ツールなどのコントロール群を除いたグリッドコンテナのcontent boxを測定する。
- `ResizeObserver`で幅と高さの変化を監視する。
- 面積予算の基準セル数は、グリッドコンテナ内に表示可能な列数と行数から求める。
- gapは利用可能領域から控除する。端数セルは利用可能セル数へ含めない。
- 高さ0、幅0、非表示など測定不能な状態では割り当てを保留し、有効寸法を得た時点で計算する。

### 7.2 制約の適用順序

表示順の先頭から、各項目へ次の順序で確定サイズを割り当てる。

1. スケーラブルでなければ等倍を選ぶ。ただし幅に収まらない場合は、収まる最大段階まで降格する。
2. スケーラブルなら分位点から希望サイズを求める。
3. グリッド幅に収まるか確認する。収まらなければ1段階降格して再確認する。
4. 対象サイズを追加しても累積面積予算内か確認する。
5. 予算を超える場合は1段階降格し、幅とその段階の予算を再確認する。
6. 条件を満たす最初の段階を確定サイズとする。
7. `1/16`は幅に1セル以上あれば予算検査なしで受け入れる。

各項目は最大6回の降格判定で確定するため、ソート済み入力に対する割り当て処理は`O(n)`とする。

### 7.3 累積面積予算

面積予算はグリッドコンテナの1ビューポート分の利用可能セル面積を基準にする。

| 対象サイズ | 累積上限 |
| --- | ---: |
| `4` | 25% |
| `4`から`2` | 45% |
| `4`から`1` | 70% |
| `4`から`1/2` | 85% |
| `4`から`1/4` | 92% |
| `4`から`1/8` | 97% |
| `4`から`1/16` | 無制限 |

- 上限はUX調整値としてロジックから分離する。
- 大きなタイル1個でも上限を超える場合、例外を設けず降格する。
- 件数割合は希望サイズの分位点、画面占有は面積予算でそれぞれ制約する。
- 全項目が1画面へ収まることは要求しない。縦スクロールを許可する。

### 7.4 非スケーラブル軸

非スケーラブル軸では原則として全件を等倍`1`にする。横幅が等倍タイルを収容できない場合に限り、横スクロールを発生させないため、幅に収まる最大段階へ全件を一律降格する。非スケーラブル時は項目ごとの順位差をサイズへ持ち込まない。

## 8. グリッド描画

- CSS Gridを使用する。
- `grid-auto-flow: dense`は使用しない。
- DOM順、視覚順、キーボード順の一致を維持する。
- 大小の組み合わせによって空きセルが生じることを許容する。
- 横スクロールは禁止し、縦スクロールのみ許可する。
- サイズはdata属性またはクラスで表現し、CSS側でspanと情報表示を切り替える。
- 初期版では再配置およびサイズ変更のアニメーションを行わない。

## 9. 再計算契機と競合制御

次の場合にサイズを再計算する。

- 初回データ取得完了
- 検索語変更
- フィルタ変更
- ソート軸変更
- ソート方向変更
- 自由移動モード変更
- 表示集合の追加、削除、更新
- グリッドコンテナの寸法変更

リサイズのみ120msのデバウンスを適用する。値は定数として分離する。

- 連続したリサイズの中間状態は保証しない。
- 最後に観測した有効寸法に対する結果だけを反映する。
- 検索やソートなどリサイズ以外の状態変更は、不要な遅延を設けず再計算する。
- 非同期処理が競合した場合、古い表示集合または古い寸法に対する結果をDOMへ反映しない。

## 10. 状態表示

グリッド領域は次の状態を排他的に表示する。

| 状態 | 表示 |
| --- | --- |
| loading | 「読み込み中」相当の最小限のテキスト |
| ready | タイルグリッド |
| empty | 「ブックマークがありません」相当のテキスト |
| error | 「読み込みに失敗しました」相当のテキスト |

- エラー詳細はコンソールへ記録する。
- エラー時の再試行UIは初期版では提供しない。
- loading、empty、errorではサイズ配分を実行しない。

## 11. 操作

- 初期版で実装する操作は左クリックだけとする。
- 左クリック時は`browser.tabs.create()`を使い、新しいアクティブタブで対象URLを開く。
- クリック処理の失敗は捕捉し、未処理のPromise rejectionを発生させない。
- キーボードその他の操作は本バージョンのスコープ外とする。

## 12. 性能要件

- 初期版の想定最大件数は2,000件とする。
- ソート済み入力に対するサイズ配分は`O(n)`とする。
- パネル内部でソートも担当する場合を含めても最大`O(n log n)`とする。
- 初期版は全件DOM描画を許容する。
- 性能テストでは実行環境、ブラウザ、件数、測定対象区間を記録する。
- 合格時間の固定値は実装環境で基準値を測定してから確定する。機能テストへ不安定な絶対時間判定を混ぜない。

## 13. 責務分割案

| 責務 | 入力 | 出力 |
| --- | --- | --- |
| 表示集合生成 | 全項目、検索、フィルタ | 対象項目配列 |
| ソート | 対象項目、軸、方向 | 安定した表示順 |
| 希望サイズ計算 | 順位、件数、分位点 | 希望サイズ |
| サイズ配分 | 希望サイズ列、グリッド寸法、予算 | GUIDと確定サイズの対応 |
| DOM描画 | 表示順、確定サイズ | タイル要素 |
| リサイズ調停 | ResizeObserver通知 | デバウンス済み再計算要求 |
| 操作 | タイルの左クリック | 新規アクティブタブ |

希望サイズ計算とサイズ配分はDOMおよびWebExtension APIから独立した純粋関数とし、TDDの中心に置く。

各フェーズは独立したGO、Red、Green、必要時のRefactor、テスト結果報告、STAYの単位とする。

## 15. テストケース

### 15.1 希望サイズ計算（単体）

| ID | 前提・入力 | 期待結果 |
| --- | --- | --- |
| PS-01 | `N=0` | サイズ計算を行わず空結果になる |
| PS-02 | `N=1, i=0` | `p=0`、希望サイズ`4`になる |
| PS-03 | `N=2` | `p=0, 0.5`となり、それぞれ`4, 1/4`になる |
| PS-04 | 各境界の直前 | 境界より大きい側のサイズになる |
| PS-05 | `p=0.05, 0.15, 0.30, 0.50, 0.70, 0.85` | それぞれ次の小さい段階へ切り替わる |
| PS-06 | `scalable=false` | 全件の希望サイズが`1`になる |
| PS-07 | 昇順のスケーラブル軸 | 昇順適用後の先頭から大きい希望サイズになる |
| PS-08 | 降順のスケーラブル軸 | 降順適用後の先頭から大きい希望サイズになる |
| PS-09 | 同値項目が複数 | 入力された表示順を維持する |
| PS-10 | スケーラブル軸に欠損値 | 欠損を最低値として並べ、方向に対応した順位になる |
| PS-11 | 検索で集合が縮小 | 縮小後の`N`と順位で全件の希望サイズを再計算する |
| PS-12 | 分位点設定を差し替える | 差し替えた境界に従い、アルゴリズム本体は変更不要 |

### 15.2 幅制約・面積予算（単体）

| ID | 前提・入力 | 期待結果 |
| --- | --- | --- |
| AL-01 | 希望サイズが幅と予算内 | 希望サイズをそのまま確定する |
| AL-02 | 希望サイズが列幅を超過 | 収まる最大段階まで降格する |
| AL-03 | サイズ`4`の1件で25%を超過 | 例外なく`2`以下へ降格する |
| AL-04 | `4`は25%内だが`4+2`が45%超過 | 対象の`2`を下位段階へ降格する |
| AL-05 | 各累積境界と同面積 | 上限内として受け入れる |
| AL-06 | 各累積境界を1セル超過 | 1段階降格して再判定する |
| AL-07 | 全予算を超過 | 最終的に無制限の`1/16`へ到達する |
| AL-08 | 最小1列だけ利用可能 | 全項目が幅に収まり横方向へはみ出さない |
| AL-09 | 幅または高さが0 | 配分を保留し、不正な除算や無限ループを起こさない |
| AL-10 | 非スケーラブルかつ等倍が幅に収まる | 全件`1`になる |
| AL-11 | 非スケーラブルかつ等倍が幅に収まらない | 全件を同じ、収容可能な最大段階へ降格する |
| AL-12 | 2,000件 | 全件に必ず1つの確定サイズを返し、入力順を維持する |
| AL-13 | 面積予算設定を差し替える | 差し替えた上限に従い、アルゴリズム本体は変更不要 |

## 17. 既知の懸念と実装前確認事項

- CSS Gridのgapを含む厳密なセル数・面積算出式は、P3のテストケース確定時に具体化する。
- サイズ`4`の192px四方が実際のFirefoxパネルでどの程度強く見えるかは、実機で調整する。
- 24pxの`1/16`は操作領域が小さい。初期版では左クリックのみとする判断を受け入れ、改善はTODOとする。
- 全件DOM描画は2,000件で計測し、許容できない場合は仮想化を別フェーズとして再計画する。
- `ResizeObserver`が返す小数ピクセルとズーム倍率を考慮し、セル数算出は安全側へ切り捨てる。
- ドッキングベイによる高さ変動が連続する場合も、リサイズの最終状態だけを反映する。
